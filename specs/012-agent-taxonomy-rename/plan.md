## Summary

Rename the `task` tool to `agent`, rename the `build` agent to `liteai`, and verify the final agent roster across the LiteAI codebase. This is a mechanical, low-risk phase that establishes the new naming convention and unblocks all subsequent phases of the Plan Mode Redesign. No backward compatibility ‚Äî clean break for v-Next major release.

## Implementation Constraints

1. **No backward compatibility**: No aliases, shims, or dual-name support (v-Next mandate)
2. **Preserve `task_id` parameter name**: The parameter within the agent tool schema stays `task_id` ‚Äî it's a generic term for "unit of work ID", not the tool name
3. **Preserve historical data**: Old session logs with `"task"` tool calls are read-only and not migrated
4. **Atomic commit**: All renames should be in a single commit to avoid broken intermediate states
5. **Permission identity change**: `"task"` ‚Üí `"agent"` in permission checks affects user config files that reference the `task` permission ‚Äî this is an intentional breaking change per v-Next policy
