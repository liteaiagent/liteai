# Telemetry & Tracing

LiteAI utilizes **OpenTelemetry (OTel)** as its foundational tracing and logging backbone, paired with **Perfetto** for local performance profiling. This guide explains how to enable, capture, and view traces generated by the application.

## 1. Enabling Telemetry

By default, telemetry is turned off to save resources. You can enable it by setting environment variables when running LiteAI.

### Master Switch
To enable the OpenTelemetry subsystem, you must set:
```bash
export LITEAI_ENABLE_TELEMETRY=1
```

### Enable Perfetto Trace Dumps (Local Timeline Profiling)
To generate local Chrome Trace Event JSON files containing deep timeline profiling (e.g., LLM request spans, tool execution durations, Time To First Token):
```bash
export LITEAI_PERFETTO_TRACE=1
```

### Example: Running with Full Tracing
```bash
LITEAI_ENABLE_TELEMETRY=1 LITEAI_PERFETTO_TRACE=1 npm run start
```

---

## 2. Viewing Perfetto Traces

If you enabled `LITEAI_PERFETTO_TRACE=1`, the system records all spans into a structured JSON file buffer and periodically flushes them to your system.

### Where are the files saved?
Traces are automatically dumped into your user directory:
- **Location:** `~/.liteai/traces/`
- **Filename Format:** `trace-<session-id>.json`

*Note: Traces auto-flush periodically and gracefully flush upon the server shutting down. If the node process crashes abruptly, the latest spans may be lost.*

### How to View the Files
You do NOT need a custom viewer or backend endpoint to view these files.

1. Open your web browser.
2. Navigate to Google's official Perfetto UI: **[https://ui.perfetto.dev](https://ui.perfetto.dev)**.
3. Click "Open trace file" on the left navigation bar, or **drag and drop** your `trace-<session-id>.json` file directly into the browser window.
4. You will be presented with a highly detailed timeline visualization showing horizontal flame graphs of the exact execution blocks (`startInteraction` ➔ `llm_request` ➔ `tool_execution` ➔ `hook`).
5. You can click on any specific block to view rich metadata payloads (such as token counts or prompt hashes) attached to those spans at the bottom of the screen.

---

## 3. Advanced: OpenTelemetry Exporters

Because the system is built on standard `sdk-trace-base`, you can easily forward span metrics or structured OTel logs to commercial observability endpoints (e.g., Honeycomb, Jaeger, Datadog) by combining the master switch `LITEAI_ENABLE_TELEMETRY=1` with standard OpenTelemetry exporter variables.

For example, to dump logs to your console stdout instead of a cloud endpoint:
```bash
export OTEL_TRACES_EXPORTER=console
export OTEL_LOGS_EXPORTER=console
```

For network-based exporters, you configure the standards-compliant endpoints:
```bash
export OTEL_EXPORTER_OTLP_PROTOCOL=http/protobuf
export OTEL_EXPORTER_OTLP_ENDPOINT=https://your-ingest-layer.com/v1/traces
```

*For more information on standard routing, please consult the [official OpenTelemetry documentation](https://opentelemetry.io/docs/languages/js/).*
